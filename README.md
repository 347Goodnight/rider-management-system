# 骑手花名册管理系统 🚴‍♂️

一个面向运营人员的骑手花名册管理 Web 应用，用于管理上海、杭州、深圳三个城市的骑手信息。支持 Excel 导入导出、骑手信息的增删改查，并按城市/站点进行筛选。

## 功能特性 ✨

- **多城市管理**：支持上海、杭州、深圳三个城市的骑手管理
- **站点管理**：预置20个配送站点，按城市分类
- **Excel导入导出**：支持批量导入导出骑手数据
- **数据筛选**：可按城市和站点组合筛选骑手
- **增删改查**：完整的骑手信息CRUD操作
- **数据校验**：手机号格式、必填字段等校验
- **统计面板**：实时显示骑手数量统计

## 技术栈 🛠️

## 快速开始 🚀

### 1. 克隆/下载项目

```bash
cd rider-management-system
```

### 2. 启动后端服务

```bash
cd backend
npm install
npm start
```

后端服务将在 http://localhost:3001 启动

### 3. 启动前端应用

打开新的终端窗口：

```bash
cd frontend
npm install
npm start
```

前端应用将在 http://localhost:3000 启动

### Excel导入格式

导入的Excel文件需满足以下要求：

1. **文件格式**：.xlsx 或 .xls
2. **第一行**：列头（必须与上述字段名完全一致）
3. **数据行**：从第二行开始
4. **必填字段**：骑手ID、姓名、手机号、部门
5. **站点校验**：部门必须是预置站点列表中的站点

## API接口文档 📚

### 获取站点列表
```
GET /api/stations
```

### 获取骑手列表
```
GET /api/riders?city=上海&station=xxx
```

### 新增骑手
```
POST /api/riders
Content-Type: application/json

{
  "riderId": "R001",
  "name": "张三",
  "phone": "13800138000",
  "department": "上海徐汇区徐家汇站-UB",
  ...
}
```

### 更新骑手
```
PUT /api/riders/:riderId
Content-Type: application/json
```

### 删除骑手
```
DELETE /api/riders/:riderId
```

### 导入Excel
```
POST /api/riders/import
Content-Type: multipart/form-data
file: <Excel文件>
```

### 导出Excel
```
GET /api/riders/export?city=上海&station=xxx
```

## 注意事项 ⚠️

1. 后端服务需要先启动，前端才能正常访问数据
2. 数据存储在 `backend/data.json` 文件中，无需额外配置数据库
3. 导入Excel时会校验手机号格式和站点有效性
4. 骑手ID是唯一的，重复导入会更新已有数据
