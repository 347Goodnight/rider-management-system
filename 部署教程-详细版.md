## 二、部署后端到 PythonAnywhere

### 步骤 1：注册账号 📝

1. 打开 https://www.pythonanywhere.com/
2. 点击绿色按钮 **"Start running Python online in less than a minute"**
3. 填写注册信息：
   - **Username**: 你的用户名（记住这个，后面会用到！）
   - **Email**: 你的邮箱
   - **Password**: 密码
4. 点击 **"Create account"**
5. 去邮箱验证账号

### 步骤 2：上传代码 📤

有两种方式，推荐 **方式 A（Git）**：

#### 方式 A：通过 Git 上传（推荐）

**先在本地操作：**

1. 打开你的项目文件夹 `rider-management-system`
2. 在文件夹内右键 → **Git Bash Here**（如果没有 Git，先安装 Git）
3. 依次执行以下命令：

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交（保存）
git commit -m "第一次提交 - Flask后端"

# 连接到你的 GitHub 仓库（先要在 GitHub 创建仓库）
git remote add origin https://github.com/你的GitHub用户名/rider-management-system.git

# 推送到 GitHub
git push -u origin main
```

**然后在 PythonAnywhere 操作：**

1. 登录 PythonAnywhere
2. 点击顶部菜单 **"Consoles"**
3. 点击 **"Bash"** 打开命令行
4. 输入以下命令：

```bash
# 克隆你的仓库
cd ~
git clone https://github.com/你的GitHub用户名/rider-management-system.git

# 进入后端文件夹
cd rider-management-system/backend_flask

# 查看文件是否都在
ls
```

#### 方式 B：直接上传文件（适合不会 Git 的同学）

1. 登录 PythonAnywhere
2. 点击顶部菜单 **"Files"**
3. 点击 **"Open another directory"**
4. 输入 `rider-management-system/backend_flask`，点击 **"Open"**
5. 点击 **"Upload a file"**
6. 逐个上传以下文件：
   - `app.py`
   - `requirements.txt`
   - `wsgi.py`

### 步骤 3：创建虚拟环境 🐍

1. 点击顶部菜单 **"Consoles"** → **"Bash"**
2. 依次执行：

```bash
# 进入后端文件夹
cd ~/rider-management-system/backend_flask

# 创建虚拟环境（Python 3.10）
python3.10 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 你会看到前面出现 (venv)，表示激活成功
# 安装依赖包
pip install -r requirements.txt

# 等待安装完成...
```

> ⏱️ 安装可能需要 2-3 分钟，请耐心等待

### 步骤 4：创建 Web 应用 🌐

1. 点击顶部菜单 **"Web"**
2. 点击 **"Add a new web app"**
3. 选择 **"Manual configuration"**（手动配置）
4. 选择 **Python 3.10**
5. 点击 **"Next"**

### 步骤 5：配置 WSGI 文件 ⚙️

1. 在 Web 配置页面，找到 **"Code"** 部分
2. 点击 **"WSGI configuration file"** 的链接（类似 `/var/www/你的用户名_pythonanywhere_com_wsgi.py`）
3. **删除里面所有内容**，替换为以下代码：

```python
import sys
import os

# 添加项目路径（把 "你的用户名" 换成你的真实用户名！）
path = '/home/你的用户名/rider-management-system/backend_flask'
if path not in sys.path:
    sys.path.insert(0, path)

# 激活虚拟环境
activate_this = '/home/你的用户名/rider-management-system/backend_flask/venv/bin/activate_this.py'
exec(open(activate_this).read(), {'__file__': activate_this})

# 导入 Flask 应用
from app import app as application
```

4. 点击 **"Save"**（绿色按钮）

### 步骤 6：配置静态文件 📁

如果需要 Excel 导出功能，需要配置上传目录：

1. 在 Web 配置页面，找到 **"Static files"** 部分
2. 点击 **"Add a new static file mapping"**
3. 填写：
   - **URL**: `/uploads/`
   - **Directory**: `/home/你的用户名/rider-management-system/backend_flask/uploads`
4. 点击 **"Save"**

### 步骤 7：重启应用 🔄

1. 返回 Web 配置页面顶部
2. 点击大大的绿色按钮 **"Reload 你的用户名.pythonanywhere.com"**
3. 等待几秒钟

### 步骤 8：测试是否成功 ✅

1. 打开浏览器
2. 访问：`https://你的用户名.pythonanywhere.com/`
3. 如果看到以下 JSON，说明成功了！🎉

```json
{
  "message": "骑手花名册管理系统 API",
  "version": "1.0.0",
  "status": "running"
}
```

4. 再测试登录接口：
   - 访问：`https://你的用户名.pythonanywhere.com/api/auth/login`
   - 应该会显示 "Method Not Allowed"（这是正常的，因为需要用 POST）

**后端部署完成！** 记下你的后端地址：
```
https://你的用户名.pythonanywhere.com/api
```

## 三、部署前端到 Vercel

### 步骤 1：修改生产环境配置 📝

1. 打开项目文件夹中的 `frontend/.env.production`
2. 把里面的内容改成：

```
VITE_API_URL=https://你的用户名.pythonanywhere.com/api
```

> 💡 把 "你的用户名" 换成你在 PythonAnywhere 的真实用户名

3. 保存文件

### 步骤 2：推送到 GitHub 📤

**方案 A：单独推送前端文件夹（推荐）**

1. 打开 `frontend` 文件夹
2. 右键 → **Git Bash Here**
3. 执行：

```bash
# 初始化
git init

# 添加文件
git add .

# 提交
git commit -m "前端代码 - 准备部署"

# 连接 GitHub（先在 GitHub 创建新仓库，比如叫 rider-frontend）
git remote add origin https://github.com/你的GitHub用户名/rider-frontend.git

# 推送
git push -u origin main
```

**方案 B：推送整个项目**

如果你之前已经推送了整个项目，也可以直接部署整个仓库，Vercel 会让你选择哪个文件夹。

### 步骤 3：部署到 Vercel 🚀

1. 访问 https://vercel.com
2. 点击 **"Sign Up"** 注册（可以用 GitHub 账号直接登录）
3. 登录后，点击 **"Add New Project"**
4. 找到你的 `rider-frontend` 仓库，点击 **"Import"**
5. 配置项目：
   - **Framework Preset**: Create React App
   - **Root Directory**: `./`（如果是整个项目，选 `frontend`）
   - **Build Command**: `npm run build`
   - **Output Directory**: `build`
6. 点击 **"Deploy"**
7. 等待 1-2 分钟...

### 步骤 4：部署成功 🎉

1. 看到 **"Congratulations!"** 就说明部署成功了！
2. Vercel 会给你一个域名，比如：
   ```
   https://rider-frontend-xxx.vercel.app
   ```
3. 点击域名访问，看看能否打开登录页面

**前端部署完成！** 记下你的前端地址。

## 四、配置 CORS 联调 🔗

现在前端和后端都部署好了，但是它们还不能通信，需要配置 CORS。

### 步骤 1：修改后端 CORS 配置

1. 打开 `backend_flask/app.py`
2. 找到这一行：
   ```python
   CORS(app, origins="*")
   ```
3. 改成：
   ```python
   CORS(app, origins=[
       "https://你的前端域名.vercel.app",  # Vercel 前端地址
       "http://localhost:3000"              # 本地开发（可选）
   ])
   ```

4. 保存文件

## 五、常见问题解决 🐛

### 问题 1：PythonAnywhere 显示 "Something went wrong"

**原因**：WSGI 配置错误

**解决**：
1. 点击 Web 页面的 **"Error log"** 查看错误信息
2. 常见错误：
   - 路径不对 → 检查 WSGI 文件中的用户名
   - 虚拟环境没激活 → 检查 `activate_this.py` 路径
   - 缺少依赖 → 重新运行 `pip install -r requirements.txt`

### 问题 2：前端登录提示 "网络错误"

**原因**：CORS 没配置好，或者 API 地址错误

**解决**：
1. 检查 `.env.production` 中的 API 地址是否正确
2. 检查 `app.py` 中的 CORS 配置是否包含 Vercel 域名
3. 重新部署前端（修改环境变量后需要重新部署）
4. 重启 PythonAnywhere

### 问题 3：Excel 导入导出失败

**原因**：缺少 openpyxl 或者上传目录不存在

**解决**：
```bash
# 在 PythonAnywhere 的 Bash 中执行
cd ~/rider-management-system/backend_flask
source venv/bin/activate
pip install openpyxl

# 创建上传目录
mkdir -p uploads
```

### 问题 4：数据丢失

**原因**：PythonAnywhere 免费版会定期清理 `/tmp` 目录

**解决**：
- 我们的数据存在项目目录下，不会被清理
- 但是建议定期备份 `data.json` 文件

### 问题 5：Git 推送失败

**错误信息**：`Failed to connect to github.com port 443`

**解决**：这是网络问题，多试几次，或者：
1. 检查是否能访问 GitHub
2. 使用 GitHub Desktop 图形界面工具
3. 直接手动上传文件到 PythonAnywhere
