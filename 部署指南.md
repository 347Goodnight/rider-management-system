## 🐍 第一步：部署 Flask 后端到 PythonAnywhere

### 1. 注册 PythonAnywhere 账号
- 访问 https://www.pythonanywhere.com/
- 点击 "Start running Python online in less than a minute"
- 注册免费账号（不需要信用卡！）

### 2. 上传代码
有两种方式：

#### 方式 A：通过 Git（推荐）
```bash
# 在本地先把代码推送到 GitHub
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/rider-management-system.git
git push -u origin main
```

然后在 PythonAnywhere 的 Bash 中：
```bash
cd ~
git clone https://github.com/你的用户名/rider-management-system.git
```

#### 方式 B：直接上传文件
1. 在 PythonAnywhere 的 Files 标签页
2. 创建文件夹 `rider-management-system`
3. 逐个上传 `backend_flask` 文件夹下的文件

### 3. 创建虚拟环境并安装依赖
在 PythonAnywhere 的 Bash 中执行：
```bash
cd ~/rider-management-system/backend_flask

# 创建虚拟环境
python3.10 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 4. 配置 Web 应用
1. 点击 PythonAnywhere 顶部的 "Web" 标签
2. 点击 "Add a new web app"
3. 选择 "Manual configuration"
4. 选择 Python 3.10
5. 点击 "Next"

### 5. 配置 WSGI 文件
1. 在 Web 配置页面，找到 "Code" 部分
2. 点击 "WSGI configuration file" 链接
3. 删除原有内容，替换为：

```python
import sys
import os

# 添加项目路径
path = '/home/你的用户名/rider-management-system/backend_flask'
if path not in sys.path:
    sys.path.insert(0, path)

# 激活虚拟环境
activate_this = '/home/你的用户名/rider-management-system/backend_flask/venv/bin/activate_this.py'
exec(open(activate_this).read(), {'__file__': activate_this})

# 导入 Flask 应用
from app import app as application
```

4. 点击 "Save"

### 6. 配置静态文件（可选）
如果需要上传文件功能，配置上传目录：
1. 在 Web 配置页面的 "Static files" 部分
2. 添加：
   - URL: `/uploads/`
   - Directory: `/home/你的用户名/rider-management-system/backend_flask/uploads`

### 7. 重启应用
点击 "Reload" 按钮重启应用

### 8. 测试 API
访问 `https://你的用户名.pythonanywhere.com/`
应该能看到：
```json
{
  "message": "骑手花名册管理系统 API",
  "version": "1.0.0",
  "status": "running"
}
```

## ⚛️ 第二步：部署 React 前端到 Vercel

### 1. 更新生产环境配置
修改 `frontend/.env.production`：
```
VITE_API_URL=https://你的用户名.pythonanywhere.com/api
```

### 2. 推送到 GitHub
```bash
cd frontend
git init
git add .
git commit -m "Frontend ready for deployment"
git remote add origin https://github.com/你的用户名/rider-frontend.git
git push -u origin main
```

### 3. 部署到 Vercel
1. 访问 https://vercel.com
2. 用 GitHub 账号登录
3. 点击 "Add New Project"
4. 导入 `rider-frontend` 仓库
5. 配置：
   - Framework Preset: Create React App
   - Build Command: `npm run build`
   - Output Directory: `build`
6. 点击 "Deploy"

### 4. 配置 CORS（重要！）
部署后，需要在 Flask 后端配置 CORS，允许 Vercel 域名访问：

修改 `backend_flask/app.py`：
```python
# 原来的
CORS(app, origins="*")

# 改为（更安全）
CORS(app, origins=[
    "https://你的前端项目名.vercel.app",
    "http://localhost:3000"  # 本地开发
])
```

然后重新上传到 PythonAnywhere 并重启。

## 🐛 常见问题

### 1. PythonAnywhere 显示 "Something went wrong"
- 检查 WSGI 文件路径是否正确
- 查看 Error log 排查错误

### 2. 前端无法连接后端
- 检查 `.env.production` 中的 API 地址
- 确认 CORS 配置正确
- 检查 PythonAnywhere 是否已 Reload

### 3. Excel 导入导出失败
- 确认安装了 `openpyxl`：
  ```bash
  pip install openpyxl
  ```

### 4. 数据丢失
PythonAnywhere 免费版会定期清理文件，建议：
- 定期备份 `data.json` 文件
- 或者升级到付费版
